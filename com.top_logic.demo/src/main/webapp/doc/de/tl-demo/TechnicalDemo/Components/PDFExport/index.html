Hier kann der PDF Export getestet werden.

Es gibt verschiedene Typen A, B, C, D (im Modul test.pdfExport) wobei A eine Oberklasse von B, C, und D ist.

  • In A ist eine Formulardefinition angegeben und eine PDF-Export Definition.
    Wenn A exportiert wird, wird die PDF-Export Definition von A benutzt.
  • In B ist eine PDF- Export-Definition gegeben.
    Wenn B exportiert wird, wird die PDF-Export Definition von B benutzt, angezeigt wird aber das Formular von A.
  • In C ist nichts konfiguriert.
    Es wird sowohl bei der Anzeige als auch beim Export die Definition von A benutzt.
  • In D ist nichts konfiguriert. In der Komponente ist eine lokale PDF-Export Definition gesetzt.
    Hier wird beim Export die konfigurierte PDF-Export Definition, an anderen Stellen würde die Definition von A benutzt werden.

Snapshots

Mit dem Druck-Befehl in der Snapshot-Tabelle kann das aktuell angzeigte Objekt in ein PDF-Dokument gedruckt werden und die so erzeugte PDF-Datei im System abgelegt werden. Die Snapshot-Tabelle zeigt alle ausgedruckten Versionen des aktuellen Objektes an. Über den PDF-Download der PDF-Datei kann nachvollzogen werden, welche Wert das Objekt zu dem Zeitpunkt der Snapshot-Erzeugung hatte.

Nebenbemerkung: Diese Funktionalität ist eine reine Demo für die PDF-Export-Funktion in TLScript. Eine nachvollziehbare Historie von Objekten könnte viel einfacher über die eingebaute Daten-Historisierung von TopLogic realisiert werden.
