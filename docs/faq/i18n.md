# FAQ: Internationalization (labels and error messages)

TopLogic renders all user-facing text from resource bundles. English text is generated at build time; other languages are hand-maintained.

## Generated vs. hand-maintained bundles

- `messages_en.properties` (in `src/main/java/META-INF/`) are **generated during `mvn install`** by the `TLDoclet` (part of the `maven-javadoc-plugin` lifecycle) from config-property names and JavaDoc. **Do not edit them by hand** — the next build overwrites them. Consequently, **never pass `-Dmaven.javadoc.skip=true` when adding or renaming `I18NConstants`**, or the doclet is skipped and the properties go stale.
- `messages_de.properties` (and other locales) are **hand-maintained**. The build seeds new keys via DeepL auto-translation but never overwrites existing entries, so correcting an awkward translation directly in the file is the normal workflow.

## Changing a UI label

Labels are derived from configuration-property names and overridable with `@Label`; tooltips come from the getter's JavaDoc.

1. Add / adjust the annotation on the config property:

   ```java
   @Name("cssClassOverride")
   @Label("CSS class override")  // overrides the auto-derived "Css class override"
   boolean getCssClassOverride();
   ```

2. Run `mvn install` on the module to regenerate the properties files.
3. Commit **both** the Java change and the regenerated `messages_*.properties`:

   ```bash
   mvn install
   git diff src/main/java/META-INF/messages_en.properties   # verify
   git add src/main/java/META-INF/messages_*.properties
   git commit -m "Ticket #XXXXX: Regenerate messages with label fixes."
   ```

## User-facing errors: TopLogicException + I18NConstants

Throw `com.top_logic.util.error.TopLogicException` for errors a user should see; use a plain `RuntimeException` for internal programming errors the user cannot act on.

Messages are `ResKey` constants declared in an `I18NConstants` class, with an `@en` JavaDoc default:

```java
import com.top_logic.util.error.TopLogicException;

public class I18NConstants extends I18NConstantsBase {
    /**
     * @en Failed to generate PDF: {0}
     */
    public static ResKey1 ERROR_PDF_GENERATION_FAILED__MSG;

    static {
        initConstants(I18NConstants.class);
    }
}

throw new TopLogicException(I18NConstants.ERROR_PDF_GENERATION_FAILED__MSG.fill(ex.getMessage()), ex);
```

**Naming:** `ERROR_<DESCRIPTION>__<PARAM1>_<PARAM2>` — e.g. `ERROR_INVALID_PAGE_SIZE__VALUE_VALID`. Pick the `ResKey` variant by parameter count: `ResKey` (0 params), `ResKey1` (1), `ResKey2` (2), … The JavaDoc must start with `@en` for the English default text.

For **model** type/attribute/classifier labels (a different bundle, under `WEB-INF/conf/resources/`), see the `tl-model` skill instead.
