TopLogic führt eine Änderungshistorie für alle Fachobjekte. Mit diesen Funktionen können die an einem Fachobjekt vorgenommenen Änderungen abgefragt und bei Bedarf zurückgenommen werden.

Eine Änderung ist ein Fachobjekt vom Typ tl.changelog:ChangeSet. Für jeden ChangeSet stehen folgende Attribute zur Verfügung (Auswahl):

  • revision – die Revision der Änderung.
  • author / date / message – Metadaten der Änderung.
  • revertedBy – wenn die Änderung zurückgenommen wurde, der ChangeSet der Rücknahme; sonst null.
  • isRevert – ob es sich um die Rücknahme einer anderen Änderung handelt.
  • isRedo – ob es sich um die Wiederherstellung einer zurückgenommenen Änderung handelt.
