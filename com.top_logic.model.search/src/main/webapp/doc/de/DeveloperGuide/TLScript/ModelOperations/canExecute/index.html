Syntax

$object.canExecute($operation)

canExecute($object, $operation)

Beschreibung

Prüft, ob der aktuell angemeldete Nutzer eine Fachoperation auf dem angegebenen Objekt ausführen darf, und liefert einen Wahrheitswert.

Die Operation wird über ihre Kommandogruppen-Id referenziert (z. B. "Approve"). Dies ist die generische Prüfung hinter Fachoperationen, die über Lesen/Schreiben/Löschen hinausgehen. Die Funktion schlägt nie fehl (außer wenn die genannte Operation nicht existiert) und filtert nicht.

Die Prüfung erfolgt für den aktuell angemeldeten Nutzer und folgt denselben Regeln wie die implizite Berechtigungsprüfung: standardmäßige Verweigerung, Rollenvererbung entlang der Sicherheitshierarchie sowie Vollzugriff für Administratoren oder Ausführungen im Systemkontext.

Parameter

NameTypTypbeschreibungPflichtStandardwert
objectFachobjektDas Objekt, auf dem die Operation ausgeführt würde.ja
operationZeichenketteDie Id der zu prüfenden Kommandogruppe (Operation), z. B. eine eingebaute Gruppe wie "Delete" oder eine eigene wie "Approve".ja

Rückgabewert

Typ: Wahrheitswert

true, wenn der aktuelle Nutzer die Operation auf dem Objekt ausführen darf, sonst false. Schlägt fehl, wenn keine Kommandogruppe mit der angegebenen Id existiert.

Beispiele

Beispiel 1

$order.canExecute("Approve")

Ausgabe: Ob der angemeldete Nutzer den Auftrag genehmigen darf.
