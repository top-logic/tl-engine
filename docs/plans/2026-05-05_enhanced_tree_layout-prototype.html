TreeLayout-Vorschläge — interaktiver Prototyp

1 · Baseline (aktueller Algorithmus)

Spalte = Tiefe. Y-Stapel pro Spalte über Vorgänger-Knoten in derselben Spalte. Jeder Kindknoten erzeugt eine eigene Reihe.

2 · Sub-Grid für hohen Fan-out

Bei N > maxPerCol ordnet der Eltern-Knoten seine Kinder in einem 2D-Raster (R Zeilen × C Spalten) an. Jede Zelle enthält einen vollständigen Unterbaum. Routing: Eltern → Bus, Bus → Reihen-Feeder, Feeder → Kinder (kreuzungsfrei).

3 · Konturbasiertes Packen

Kein „Spalten-Vorgänger" mehr. Jeder Subtree trägt obere/untere Kontur pro Tiefen-Offset; Geschwister-Subtrees werden so dicht wie möglich gestapelt.

4 · Kombiniert (Sub-Grid + Kontur)

Fan-out wird als Sub-Grid aufgelöst; innerhalb jeder Zelle und für nicht-Fan-out-Knoten greift Kontur-Packen. Maximale Höhenreduktion.

Wurzelknoten orange. Bus-Connector als rechtwinkliger Pfad. Bei Sub-Grid: Eltern → vertikaler Bus → Reihen-Feeder (oberhalb jeder Zellenreihe) → kleines „L" (senkrecht hinab im Spalt links der Spalte, dann waagerecht in die linke Kante des Kindknotens). Damit treten alle Kanten konsistent horizontal aus Knoten heraus und in Knoten hinein — einheitliche Flussrichtung wie bei einem Materialfluss.
