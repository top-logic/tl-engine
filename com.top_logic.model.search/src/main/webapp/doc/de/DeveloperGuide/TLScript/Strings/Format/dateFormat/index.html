Syntax

	dateFormat($pattern, $timeZone)

Beschreibung

Konstruiert ein Datumsformat mit dem angegebenen Muster für entweder parse() oder format() Aufrufe.

Siehe SimpleDateFormat.

Standardmäßig wird der Wert in der Standard-Zeitzone der JVM dargestellt. Wird eine timeZone angegeben, wird der Wert stattdessen in dieser Zeitzone dargestellt. Das ist beim Formatieren eines Kalender-Wertes relevant: Übergibt man dessen Zeitzone, wird der Zeitpunkt als der Tag dargestellt, den er in dieser Zeitzone repräsentiert. So wird eine Verschiebung um Tag/Monat/Jahr vermieden, wenn sich die System-Zeitzone von der Standard-Zeitzone der JVM unterscheidet.

Parameter

Name Typ Beschreibung Pflicht Default
pattern Zeichenkette

Das Pattern in dem formatiert werden soll. Folgende Zeichen sind möglich:

  • y – Jahr
  • M - Monat im Jahr
  • d - Tag im Monat
  • für weitere Symbole siehe SimpleDateFormat.
 ja
timeZone Zeichenkette / Fachobjekt

Die Zeitzone, in der der Wert dargestellt werden soll. Zulässig sind eine Zeitzonen-ID als Zeichenkette (z. B. "Asia/Tokyo"), die Zeichenkette "system" für die konfigurierte System-Zeitzone der Anwendung, die Zeichenkette "user" für die Zeitzone des aktuellen Benutzers oder die Zeitzone eines Kalenders.

 nein Standard-Zeitzone der JVM

Rückgabewert

Typ: Fachobjekt

Ein Datumsformat, das nun für parse() oder format() Aufrufe verwendet werden kann.

Beispiele

Format

	dateFormat("dd-MM-yy").format(date(2012, 11, 5))

Ausgabe: 05-12-2012

Parse

	dateFormat("dd-MM-yy").parse("04-12-12")

Ausgabe: 04.12.2012

Einen Kalender in seiner eigenen Zeitzone formatieren

	cal = date(2025, 11, 1).toSystemCalendar();
dateFormat("yyyy-MM-dd", $cal.timeZone()).format($cal.toDate())

Ausgabe: 2025-12-01
