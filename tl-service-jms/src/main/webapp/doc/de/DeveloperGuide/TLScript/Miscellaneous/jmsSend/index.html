Syntax

	jmsSend($destination, $message)

Beschreibung

Die Funktion dient dazu Nachrichten an eine Warteschlange (Queue) oder ein Thema (Topic) zu senden. Hierfür wird die Java-interne JMS API (Java Message Service / Jakarta Messaging) verwendet, die es ermöglicht Nachrichten an einen Server zu senden und synchron oder asynchron vom Server abzufragen.

Damit eine Nachricht gesendet werden kann, muss eine Ziel-Konfiguration im JMS-Service angelegt sein. Der in der Konfiguration definierte Warteschlangenmanager muss unter der angegebenen Adresse erreichbar sein, um Erfolg beim Senden zu haben, andernfalls wird ein Fehler ausgegeben. Als Server kann hier die IBM MQ verwendet werden. Diese muss separat angelegt und gestartet worden sein.

Alle Ziel-Konfigurationen können, sofern der Nutzer über entsprechende Berechtigungen verfügt, unter Administration > Development > Service Editor > JMS Service angesehen, angelegt, bearbeitet und gelöscht werden.

Parameter

Name Typ Beschreibung Pflicht Default
destination Zeichenkette (Ziel-Konfiguration) Name einer Ziel-Konfiguration im JMS-Service ja -
message Zahl/Zeichenkette/Binärdaten/XML Nachricht, die an eine Warteschlange (Queue) oder an ein Thema (Topic) geschickt werden soll ja -

Rückgabewert

null - Da beim Senden einer Nachricht keine Rückgabe erfolgt.

Beispiele

String Nachricht (Topic)

	jmsSend("Topic1", "Hello Subscribers!");

Neue Nachricht (im Topic des Warteschlangenmanagers): Hello Subscribers!

Dieser Ausdruck sendet die Nachricht Hello Subscribers! an das Thema, das in der Konfiguration mit dem Namen "Topic1" festgelegt ist.

​

String Nachricht (Queue)

	jmsSend("Queue1", "Hello World!");

Neue Nachricht (in der Queue des Warteschlangenmanagers): Hello world!

Dieser Ausdruck sendet die Nachricht Hello World! an die Warteschlange, die in der Konfiguration mit dem Namen "Queue1" festgelegt ist.

​

XML Nachricht (Queue)

	jmsSend("Queue1", {{{<div>Hello World!</div>}}});

Neue Nachricht (in der Queue des Warteschlangenmanagers): <div>Hello World!</div>

Dieser Ausdruck sendet den XML Ausdruck <div>Hello World!</div> an die Warteschlange, die in der Konfiguration mit dem Namen "Queue1" festgelegt ist.

Binärdaten Nachricht (Queue)

	jmsSend("Queue1", binary(
    name: "hello.txt",
    data: "Hello Wörld!",
    contentType: "text/plain",
    encoding: "iso-8859-1"
));

Neue Nachricht (in der Queue des Warteschlangenmanagers): Binärnachricht

Dieser Ausdruck sendet die Binärdaten binary(name: "hello.txt", data: "Hello Wörld!", contentType: "text/plain", encoding: "iso-8859-1") an die Warteschlange, die in der Konfiguration mit dem Namen "Queue1" festgelegt ist.
