Manchmal ist es nicht ausreichend, nur die Haupt-Layout XML Datei zu verwenden, um das Layout einer spezifischen Anwendung oder Sicht zu bestimmen. Dies kann beispielsweise der Fall sein, wenn eine Anwendung eine angepasste Darstellung oder zusätzliche Komponenten benötigt, die nur für diese Anwendung oder ein spezifisches Modul relevant sind.

Funktionsweise von Layout Overlay

In einer *.layout.overlay.xml -Datei spielt die Parent-Child-Beziehung keine Rolle. Stattdesssen werden in dieser Datei Komponenten der anzupassenden Sicht über ihren lokalen Namen referenziert. Der Inhalt dieser Datei stellt eine inkrementelle Konfiguration dar, die Änderungen oder Ergänzungen zu bestehenden Komponenten beschreibt.

Anwendungsfall: Anpassung der modellbasierten Suche

Ein Beispiel ist die modellbasierte Suche, die im Modul Element definiert wird. Möchte man in einem zusätzlichen Modul, beispielsweise einem Demo-Modul, eine abweichende Version dieser Suche implementierten, etwa durch das Hinzufügen einer Komponente mit spezifischen Informationen, kann eine *.layout.overlay.xml -Datei erstellt werden.

Angenommen, man möchte im Demo-Modul die modellbasierte Suche verändern, indem zusätzliche Spalten definiert und der Title-Key einer Komponente geändert werden. Dies könnte in einer Datei wie com.top_logic.element/search/searchinput.layout.overlay.xml so aussehen:

<components>
   <component name="SearchAttributedSearchInput" columns="columns(com.top_logic.reporting/search/searchResult.layout.xml#SearchAttributedSearchResultTable)"
   />
   <securityLayout name="searchInput"
      titleKey="pos.search.input.tabber"
   />
</components>
