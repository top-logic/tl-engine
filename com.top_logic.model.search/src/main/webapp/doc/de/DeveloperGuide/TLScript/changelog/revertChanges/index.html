Syntax

	revertChanges($changes)

Beschreibung

Nimmt die übergebenen Änderungen (tl.changelog:ChangeSet) innerhalb der bereits geöffneten Transaktion zurück. Die Reihenfolge innerhalb der übergebenen Liste ist unerheblich: Die Funktion sortiert intern absteigend nach Revision und nimmt die jüngste Änderung zuerst zurück.

Die Funktion öffnet selbst keine Transaktion und führt kein Commit aus. Das umgebende Skript ist für das Commit- und Rollback-Handling verantwortlich.

Zurückgeliefert wird die Liste der Probleme (Ressourcenschlüssel), die während der Rücknahme aufgetreten sind. Eine leere Liste bedeutet, dass alle Rücknahmen ohne Konflikt ausgeführt wurden.

Parameter

Name Typ Beschreibung Pflicht Default
changes Liste von tl.changelog:ChangeSet Die Änderungen, die zurückgenommen werden sollen. Kann auch ein einzelner ChangeSet sein. ja

Rückgabewert

Typ: Liste von Ressourcenschlüsseln

Liste der während der Rücknahme aufgetretenen Probleme; leer bei fehlerfreier Ausführung.

Beispiele

Beispiel 1

	revertChanges($project.changeLog())

Setzt ein Projekt inklusive aller Unterobjekte auf den Zustand zurück, den es vor der ersten in der Historie enthaltenen Änderung hatte. Das Skript, das diesen Ausdruck enthält, muss die Änderung anschließend committen.
