In TopLogic gibt es die Möglichkeit MBeans zu erstellen, die einen TL-Script Ausdruck beinhalten, der mit einer beliebigen Anzahl an Parametern angereichert werden und ausgewertet werden kann. So können InApp MBeans erstellt und konfiguriert werden.

Über den Service "TLDataBeanService" im Serve-Editor können "MBean mit einem TL-Script-Ausdruck" (MonitorByExpression) ausgewählt werden. Dieser muss wie in Tabelle 1 zu sehen konfiguriert werden.

Name Typ Beschreibung
Name String Eindeutiger Name des MBeans nach dem Muster "domain:key=property".
Beschreibung String Beschreibungstext, der an der Oberfläche zu diesem MBean angezeigt werden soll.
Implementierung Expr TL-Script Ausdruck, der ausgewertet werden soll.
Liste von Parametern Parameter-Liste

Es müssen genauso viele Parameter erstellt werden, die für den TL-Script Ausdruck benötigt werden. Sie werden in genau der Reihenfolge an den TL-Script-Ausdruck weitergegeben.

Tabelle 1: Konfiguration eines MBeans mit einem TL-Script-Ausdruck.

Jeder benötigte Parameter muss erstellt und wie in Tabelle 2 zu sehen konfiguriert werden.

Name Typ Beschreibung
Name String Der Parameter benötigt einen Namen, der jedoch irrelevant für den TL Script Ausdruck ist.
Beschreibung String Beschreibungstext, der an der Oberfläche zu diesem Parameter angezeigt werden soll.
Typ Boolean / Number / String Der Typ des Parameters.

Tabelle 2: Konfiguration eines Parameters.

Nun können diese Parameter z. B. über JConsole befüllt werden. Dazu wird eine execute()-Operation angeboten, die auch das Ergebnis zurückliefert. Dieses Ergebnis wird auch gleichzeitig im Attribut result gespeichert.
