Syntax

$parent.canCreate($reference)

canCreate($parent, $reference)

Beschreibung

Prüft, ob der aktuell angemeldete Nutzer ein neues Element in der angegebenen Kompositionsreferenz des angegebenen übergeordneten Objekts anlegen darf, und liefert einen Wahrheitswert.

Dies entspricht dem Schreibzugriff auf den Kompositionskontext (dieselbe Prüfung, die das Hinzufügen eines Elements zur Referenz absichert). Die Funktion schlägt nie fehl und filtert nicht.

Die Prüfung erfolgt für den aktuell angemeldeten Nutzer und folgt denselben Regeln wie die implizite Berechtigungsprüfung: standardmäßige Verweigerung, Rollenvererbung entlang der Sicherheitshierarchie sowie Vollzugriff für Administratoren oder Ausführungen im Systemkontext.

Parameter

NameTypTypbeschreibungPflichtStandardwert
parentFachobjektDas übergeordnete Objekt, das das neue Element aufnimmt.ja
referenceAttributDie Kompositionsreferenz (TLStructuredTypePart), in die das Element angelegt wird.ja

Rückgabewert

Typ: Wahrheitswert

true, wenn der aktuelle Nutzer ein Element in der angegebenen Referenz des übergeordneten Objekts anlegen darf, sonst false.

Beispiele

Beispiel 1

$project.canCreate(`my.module:Project#milestones`)

Ausgabe: Ob der angemeldete Nutzer dem Projekt einen Meilenstein hinzufügen darf.
