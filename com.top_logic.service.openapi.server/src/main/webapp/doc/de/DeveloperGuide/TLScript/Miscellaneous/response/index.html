Syntax

response(
  statusCode: $statusCode, 
  content: $content, 
  contentType: $contentType
)

Beschreibung

Erstellt ein Antwort-Objekt, dass auf eine HTTP-Anfrage zurückgegeben wird.

Parameter

Name Typ Beschreibung Pflicht Default
statusCode Zahl Status-Code für die Antwort. Hier werden HTTP-Response Codes angegeben, z.B. 200 (OK) oder 404 (NOT FOUND). ja
content
Objekt
Inhalt der Antwort. Ist kein Inhalt gegeben wird der Status-Code als Fehlercode interpretiert und die Standardantwort des Containers gesendet. nein
contentType Zeichenkette
Inhaltstyp der Antwort. Dieser muss zum Resultat der Antwort passen.
nein application/json

Beispiele

Beispiel 1

response(200, {"name": "Rüdiger"});

​Die Anfrage war erfolgreich und {"name": "Rüdiger"} wird mit dem Inhaltstyp application/json übertragen.

Beispiel 2

response(404);

Es wird dDie Standard-Antwort gesendet, dass die Resource nicht gefunden wurde.

Beispiel 3

response(
  500,
  "<html>
    <head><title>Error 500 Server Error</title></head>
    <body><h2>HTTP ERROR 500 Server Error</h2></body>
   </html>",
  "text/html");

Der Fehlercode 500 wird gesendet mit dem gegebenen Inhalt.
