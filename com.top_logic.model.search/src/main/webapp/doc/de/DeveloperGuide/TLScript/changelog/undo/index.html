Syntax

	$object.undo([maxEntries[, includeSubtree]])
	undo()

Beschreibung

Nimmt innerhalb der aktuellen Transaktion die neueste „echte" Änderung zurück, die im betrachteten Fenster der Änderungshistorie gefunden wird. „Echt" bedeutet: Die Änderung wurde noch nicht zurückgenommen und ist selbst keine Rücknahme.

Die Funktion öffnet selbst keine Transaktion und führt kein Commit aus. Das umgebende Skript ist für Commit und Rollback verantwortlich.

Die Parameter entsprechen denen von changeLog und bestimmen, welches Fenster der Änderungshistorie betrachtet wird.

Parameter

Name Typ Beschreibung Pflicht Default
object Fachobjekt Der Unterbaum-Wurzel; null bedeutet die globale Anwendung. nein null
maxEntries Ganzzahl Begrenzt die Anzahl der betrachteten Historien-Einträge. Ein Wert kleiner oder gleich 0 bedeutet keine Begrenzung. nein 0
includeSubtree Wahrheitswert Legt fest, ob der komplette Kompositions-Unterbaum berücksichtigt wird oder nur das Objekt selbst. Ignoriert bei fehlendem object. nein true

Rückgabewert

Typ: Liste von Ressourcenschlüsseln

Liste der während der Rücknahme aufgetretenen Probleme; leer bei fehlerfreier Ausführung oder wenn keine rücknehmbare Änderung gefunden wurde.

Beispiele

Beispiel 1

	$project.undo()

Nimmt die neueste echte Änderung im Kompositions-Unterbaum des Projektes zurück.

Beispiel 2

	undo(null, 50)

Nimmt die neueste echte Änderung der gesamten Anwendung zurück, indem die letzten 50 Einträge der globalen Historie betrachtet werden.
