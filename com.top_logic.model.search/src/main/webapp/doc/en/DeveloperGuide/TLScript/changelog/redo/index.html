Syntax

	$object.redo([maxEntries[, includeSubtree]])
	redo()

Description

Re-applies the most recent undo performed within the considered change-log window. The revert ChangeSet with the highest revision number is itself reverted, which restores the original change.

The function does not open or commit a transaction. Commit and rollback handling is the responsibility of the surrounding script.

The parameters mirror those of changeLog and determine which window of the change history is inspected.

Parameter

Name Type Description Mandatory Default
object Business object The subtree root; null operates on the global application. no null
maxEntries Integer Upper bound on the number of change-log entries considered. A value less than or equal to 0 means unlimited. no 0
includeSubtree Boolean Whether the entire composition subtree is considered or only the object itself. Ignored when object is absent. no true

Return value

Type: List of resource keys

List of problems raised during the redo; empty when the redo ran cleanly or when no revertable undo was found.

Examples

Example 1

	$project.redo()

Re-applies the undo most recently performed on the project.
