TreeLayout-Vorschläge — interaktiver Prototyp

1 · Baseline (aktueller Algorithmus)

Spalte = Tiefe. Y-Stapel pro Spalte über Vorgänger-Knoten in derselben Spalte. Jeder Kindknoten erzeugt eine eigene Reihe.

2 · Sub-Grid für hohen Fan-out

Bei N > maxPerCol verteilt der Eltern-Knoten seine Kinder spaltenweise (R Kinder pro Spalte, C = ⌈N/R⌉ Spalten). Jede Spalte wird unabhängig vertikal gepackt; jede Spalte hat ihren eigenen Bus. Folge-Spalten-Busse hängen über eine gemeinsame Bottom-Brücke am Primär-Bus.

3 · Konturbasiertes Packen

Kein „Spalten-Vorgänger" mehr. Jeder Subtree trägt obere/untere Kontur pro Tiefen-Offset; Geschwister-Subtrees werden so dicht wie möglich gestapelt.

4 · Kombiniert (Sub-Grid + Kontur)

Fan-out wird als Sub-Grid aufgelöst; innerhalb jeder Zelle und für nicht-Fan-out-Knoten greift Kontur-Packen. Maximale Höhenreduktion.

Wurzelknoten orange. Bus-Connector als rechtwinkliger Pfad. Bei Sub-Grid: Eltern → Primär-Bus (Spalte 0) mit horizontalen Anschlüssen in die linken Kanten der Spalte-0-Kinder. Unterhalb der tiefsten Spalte verläuft eine waagerechte Bottom-Brücke, an der die Busse der Folge-Spalten von unten ansetzen und nach oben verlaufen. Alle Kanten treten konsistent horizontal in Knoten ein und aus — einheitliche Flussrichtung wie bei einem Materialfluss; die Reihenfolge der Anschlüsse pro Spalte entspricht der Originalreihenfolge der Kinder.
